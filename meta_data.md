# MyDB Meta Data

`params` are generated by the `general_form` from
  user input. This data is used to create a container and is then stored in `admin_db`.
After the container is generated meta data is stored as JSON in the `Info` field in
the meta data.

### params keys

  Core Container/Database Information:

  - Name - Alternative name field (new to version 2). `Name` must be unique, the 
    *dbname* is not unique. (projectA, foo
  - dbname - Database/container name
  - dbtype - Database type (Postgres, MongoDB, MariaDB, Neo4j)
  - dbuser - Database username
  - dbuserpass - Database user password
  - username - Real username from AD. dbuser can be different 

  Container Configuration:

  - image - Docker image to use
  - port - Port number
  - volume_name - Mapped docker swarm volume name
  - config_name - mapped docker config file name
  - command - Docker command to run

  Metadata/Labels:

  - owner - Owner name
  - contact - Contact email
  - support - Support information
  - description - Container description
  - manager - Manager of user (from AD)
  - department - Infomation from AD 
  - life - Container lifecycle/lifetime DEPRECATED
  - backup_freq - Backup frequency
  - backup_window - Backup time window DEPRECATED
  - maintain - Maintenance flag DEPRECATED
  - phi - PHI (Protected Health Information) flag DEPRECATED

  Backup/Restore Related:

  - backup_type - Type of backup (e.g., 'User')
  - backup_prefix - Prefix for backup files
  - source_container - Source container for migration
  - dest_container - Destination container for migration
  - source_dbname - Source database name
  - container_name - Container name
  - Name - Alternative name field

  MariaDB Specific:

  - EAR - Encryption At Rest flag (Yes/No)

  Test/Misc:

  - con_name - Alternative container name field
  - result - Result of operation

  The most commonly used keys across the codebase are: dbname, dbtype, dbuser, dbuserpass, port, image, db_vol,
  username, owner, contact, description, life, backup_freq, backup_window, maintain, phi, support, and
  port_bindings.
